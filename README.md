# WFGY Ecosystem

This repository is the public ecosystem hub around **WFGY**.

It exists to make the broader WFGY landscape easier to navigate, especially for people who want to understand:

- what WFGY includes
- which pages are the main entry points
- where ecosystem proof and recognition live
- how teams can explore collaboration, pilots, or integration

This repo is not the core engine itself.

The main project lives here:  
[WFGY](https://github.com/onestardao/WFGY)

---

## What this repository is for

The WFGY ecosystem now spans multiple layers:

- core engines
- problem maps
- troubleshooting surfaces
- TXT-based apps
- onboarding pages
- external proof and recognition

This repository helps organize those layers into one public navigation hub.

It also serves as the home for future ecosystem-facing documents such as adoption evidence, practical case evidence, and collaboration paths.

## How to read the ecosystem

A simple way to understand WFGY is to think of it in layers:

### 1. Engine
This is the reasoning core and tension framework behind the system.

### 2. Map
This is where failure modes, routing logic, and debug surfaces are made visible.

### 3. App
This is where WFGY becomes usable through TXT-first interfaces and modular tools.

### 4. Proof
This is where external validation, recognition, and ecosystem signals accumulate.

This repository sits across those layers and helps connect them.
